# 面包师的三色笔记

> **对应概念：** 参数（Parameter）、指标（Metric）、Tag

## 故事

老王在榆树街开了家书店，就在小梅的面包坊对面。小梅开始做巧克力蛋糕实验笔记之后，带着笔记本来找老王帮忙。问题一目了然：她的笔记一团糟。原料分量和口感评分混在一起，中间还夹着「这个可以放到节日菜单里」之类的提醒。老王完全分不清哪些是烤之前就决定好的，哪些是烤完之后量出来的。

于是老王给了她三支彩色笔和一叠便利贴。

**蓝色笔记录烤之前做的选择**——她的参数。可可比例：黑巧和牛奶巧七三开。烤箱温度：163 度。烘烤时间：28 分钟。这些是输入，烤箱门关上之前就锁定了。

**红色笔记录烤完之后量出来的结果**——她的指标。湿润度：8.2/10。膨胀高度：4.1 厘米。试吃评分：9.0/10。有时候她还会在多个时间点量（10 分钟、20 分钟、30 分钟的内部温度）——一条指标历史曲线，能看到蛋糕是怎么变化的。

**绿色便利贴记录标签**——她的 Tag。「周三最佳」「太干——跳过」「节日候选」。这些不影响配方也不影响结果，纯粹是方便以后查找分类用的。

一个星期下来，小梅翻到笔记本里任何一个 Run，一眼就能看清：放了什么（蓝色）、出了什么（红色）、怎么分类的（绿色）。镇上搞年度烘焙大赛的罗莎后来也采用了同样的红色笔系统，让评委们用统一标准给参赛者打分。

## 概念解读

**参数**是 Run 开始前设好的配置值——学习率、batch size、优化器选择、模型深度等。参数的 key 和 value 都是字符串。设好了就不变。

**指标**是数值型的结果——准确率、损失值、F1 score、延迟等。它回答的是「这个 Run 跑得怎么样」。指标可以在训练过程中反复记录（比如每个 epoch 记一次准确率），形成一条随 step 变化的曲线。

**Tag** 是额外的元数据，用来搜索和过滤——比如 owner、数据集版本、Git commit、运行环境等。纯粹是给人看的，不影响任何计算。

::: tip 一句话总结
- **参数** = 配置值（蓝色），Run 前设好，不变
- **指标** = 数值结果（红色），可以按 step 记录形成曲线
- **Tag** = 额外备注（绿色），用来搜索过滤
:::

## 前端开发者参考

你做的 **Run 表格**里的列就是这三种颜色：参数列、指标列、Tag 列。**指标图表**画的就是红色数字随 step 的变化曲线。**搜索栏**让用户按这三类任意一种来过滤。

| 组件 | 对应什么 |
|------|---------|
| Run 表格列（ag-grid） | 参数、指标、Tag 各占一类可排序列 |
| `RunViewMetricCharts` | 指标值按 step 绘制的折线图（「历史」） |
| 搜索/过滤栏 | 按参数值、指标范围、Tag 来筛选 Run |
| Run 详情面板 | 一个 Run 的全部参数、指标和 Tag |

### 数据长什么样

```typescript
// 参数——key-value，都是字符串
{ key: "learning_rate", value: "0.001" }

// 指标——数值，带 step 和毫秒级 timestamp，可以画曲线
{ key: "accuracy", value: 0.95, step: 100, timestamp: 1700000000000 }

// Tag——key-value 标记
{ key: "team", value: "nlp" }
```

::: info 为什么参数值是字符串？
因为参数可能是数字、文件路径、JSON 配置或者模型名称——什么都有。存成字符串最通用，UI 原样展示就好。而指标一定是数字——因为你要画图、要排序、要比大小。
:::
